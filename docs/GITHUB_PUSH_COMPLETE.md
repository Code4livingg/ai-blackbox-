## GitHub Workflow

### For Contributors

1. **Fork Repository** (if not a collaborator)
   - Click "Fork" on GitHub

2. **Clone Your Fork**
   ```bash
   git clone https://github.com/your-username/ai-blackbox-.git
   ```

3. **Create Feature Branch**
   ```bash
   git checkout -b feature/your-feature
   ```

4. **Make Changes**
   ```bash
   git add .
   git commit -m "feat: Your feature"
   ```

5. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature
   ```

6. **Create Pull Request**
   - Go to original repository
   - Click "New Pull Request"
   - Select your fork and branch
   - Add description
   - Submit PR

7. **Wait for Review**
   - Maintainers will review
   - Make requested changes if needed
   - PR will be merged

### For Maintainers

1. **Review Pull Requests**
   - Check code quality
   - Run tests
   - Request changes if needed

2. **Merge Pull Requests**
   ```bash
   git checkout main
   git pull origin main
   git merge feature/branch-name
   git push origin main
   ```

3. **Create Releases**
   ```bash
   git tag v1.0.0
   git push origin v1.0.0
   ```

## Troubleshooting

### Issue: "fatal: remote origin already exists"
**Solution**: Remove existing remote:
```bash
git remote remove origin
git remote add origin https://github.com/Code4livingg/ai-blackbox-.git
```

### Issue: "Permission denied (publickey)"
**Solution**: Set up SSH keys:
```bash
ssh-keygen -t ed25519 -C "your.email@example.com"
# Add public key to GitHub
```

### Issue: "Your branch is ahead of 'origin/main'"
**Solution**: Push changes:
```bash
git push origin main
```

### Issue: "Merge conflict"
**Solution**: Resolve conflicts manually:
```bash
# Edit conflicted files
git add .
git commit -m "Resolve merge conflict"
git push origin main
```
